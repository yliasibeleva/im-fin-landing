## Этап 1 — Max-бот (buh-bot/)

### Стек
- Python 3.11+, maxapi (polling), SQLite, APScheduler, openpyxl

### Что умеет
- **Компании:** добавление/редактирование, ИНН, СНО (ОСН/УСН/ЕСХН/ПСН), тип (ООО/ИП), сотрудники, воинский учёт, стандарт работы, привязка группы Max
- **Дедлайны:** автогенерация по налоговому календарю РФ при добавлении компании, произвольные дедлайны, отметка выполнения
- **Задачи:** постановка бухгалтеру, приоритет, срок, статус
- **Доп. работы:** фиксация (тип, часы, сумма) сверх абонплаты
- **Ошибки:** журнал ошибок бухгалтеров → влияет на KPI
- **Напоминания:** ежедневно в 9:00 МСК — за 14/7/3/1 день в группу клиента и бухгалтеру лично
- **Отчёты:** текстовые + Excel (доп. работы + KPI бухгалтеров)

### Роли
| Роль | Как определяется |
|------|-----------------|
| Руководитель | max_user_id в ADMIN_IDS (.env) |
| Бухгалтер | Запись в таблице accountants |
| Клиент | Участник группы в Max |

### Ключевые команды
```
/start          — главное меню
/add_company    — добавить компанию (9-шаговый диалог)
/edit_company   — редактировать компанию
/company_info   — карточка компании
/add_accountant — добавить бухгалтера
/add_task       — поставить задачу
/add_deadline   — произвольный дедлайн
/add_error      — ошибка бухгалтера
/add_work       — доп. работа
/upcoming       — дедлайны на 30 дней
/myid           — узнать ID (пользователя или группы)
/status         — статус компании (в группе клиента)
```

### Таблицы БД
- `companies` — клиенты
- `accountants` — бухгалтеры
- `report_deadlines` — дедлайны (автогенерация + ручные)
- `tasks` — задачи
- `additional_works` — доп. работы
- `accountant_errors` — журнал ошибок
- `reminder_log` — лог отправленных напоминаний (защита от дублей)

### Запуск
```bash
cd buh-bot
pip install -r requirements.txt
python main.py
```
Токен бота: `id5753064130_bot` (ImFinBot в Max)  
После запуска: написать боту `/myid` → вставить ID в `.env` как `ADMIN_IDS`

## Важные ограничения maxapi

- Только **один** `@dp.message_created` для текстов (не команд) — нельзя вешать несколько
- Только **один** `@dp.message_callback` — все callback'и обрабатываются в одном хендлере
- Polling режим (не webhook)
- Все DB-вызовы: `await asyncio.to_thread(db.func, args)`
